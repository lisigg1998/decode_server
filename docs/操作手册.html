网站迁移

如果服务器的IP发生变化，您必须进入Linux虚拟机环境修改Nginx配置并替换SSL证书
在bash环境下执行下列命令
注意：
请务必以实际IP替换两处10.20.20.12后执行
命令的其他部分（包括空格）必须保持原样以确保执行成功
建议先复制命令，再修改后复制粘贴到虚拟机中执行

sed -i "s/^    server_name.*/    server_name 10.20.20.12;/g" \
    ~/decode_website/deployment_configs/nginx_decode
cd ~/decode_website/certs/
openssl req -x509 -newkey rsa:4096 -sha256 -keyout key.pem -out cert.pem -days 3650  -nodes \
-subj "/C=CN/ST=Guangdong/L=Shenzhen/O=Shenzhen Research Institute of Big Data/OU=Learning Analytics/CN=10.20.20.12"
sudo systemctl restart nginx

日常管理

访问https://10.20.20.12
在弹出的登录对话框输入管理员用户名cuhkszadmin和密码进入管理界面
点击“Details”以查看每个项目的详细说明

第一部分更改管理员和API接口的密码

API接口密码应定期修改并通知API使用方
Admin Portal项对应访问本界面的管理员密码

第二部分管理访问学号解密网页的用户名和密码

当前已创建的所有用户名列表可在“Details”中查看
使用“Add or modify”选项添加一个新用户或修改现有用户的密码
使用“Delete”选项删除一个已创建的用户
修改密码或删除用户不需要提供他们的当前密码
注意： 此部分的用户只用于访问本网站的学号解密网页部分，与ICUHK和Learning Analytics网页无关

第三部分管理储存明文和密文学号对应表的数据库

这些数据有两个用途：

  • ICUHK用户的用户名若出现在明文学号中，将被授权访问Learning Analytics网站
  • 本网站的学号解密网页部分将使用这些数据解密本科生密文学号

第一个项目以上传的CSV文件为数据来源更新数据库内容：

  • 如果CSV文件中的一个明文学号不存在于数据库中，系统记录中将添加这个明文学号和对应的密文学号
  • 如果CSV文件中的一个明文学号已存在于数据库中，系统记录中将会用上传的密文学号代替当前数据库中的密文学号
  • 数据库中已有但CSV文件中没有的明文学号不受影响

数据要求：

  • CSV不能有标题行
  • 对于本科生，CSV第一列为明文学号（同时也是他们的@link.cuhk.edu.cn邮箱的用户名部分），第二列为对应的加密过的学号
  • 对于教授和其他管理人员，第一列是他们@cuhk.edu.cn邮箱的用户名部分，第二列为用户权限前缀+第一列
  • 目前权限前缀包括staff_和admin_。除非必要，请避免分配admin_权限，一般为分配staff_权限即可

第二个项目以输入的内容为数据来源更新数据库内容
“real ID”即明文学号（CSV第一列）， “encoded ID”即对应的密文学号（CSV第二列）

“Add new entry or modify existing entry”选项和上传CSV功能相同（创建新的条目或更新已有条目）
“Delete existing entry”选项从数据库中删除输入的明文学号和对应的密文学号

第三个项目清空储存明文和密文学号对应表的数据库
点击“DELETE ALL”会不可逆的删除所有条目
网页将弹出窗口要求确认操作：点击“取消”以取消操作，点击“确定”清空数据库
