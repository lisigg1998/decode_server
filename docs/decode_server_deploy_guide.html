Decode Server Deploy Guide

Assume that we start from a fresh installation of Ubuntu 16.04 LTS server edition.

Notes:

  1. Please use itso as your username. If not, you need to change many other configurations.
  2. There is a timeout setting in the ExecStart section of ~/decode_website/deployment_configs/gunicorn.service. It must be large enough to handle large upload files.
  3. Always start a screen / tmux session immediately after you sshed into the system to avoid problems caused by network failures.
  4. Never interrupt an apt or apt-get session when it is installing, upgrading, or removing packages.
  5. If you see that apt upgrade is going to install a new version of Linux Kernel and Firmware, make sure that /boot has enough space.

Sever Migration

If you are just migrating a server, then only two changes are necessary:

  • Change FQDN or server IP

    cd
sudo vi ~/decode_website/deployment_configs/nginx_decode

    Find two lines beginning with server_name and replace the FQDN / IP with the correct one.

  • Change SSL certificate
    Both files should be PEM format.
    Replace ~/decode_website/certs/cert.pem with certificate (public key) file.
    Replace ~/decode_website/certs/key.pem with private key file.
    To generate a self-signed SSL certificate, use

    cd ~/decode_website/certs
openssl req -x509 -newkey rsa:4096 -sha256 -keyout key.pem -out cert.pem -days 3650  -nodes \
-subj "/C=CN/ST=Guangdong/L=Shenzhen/O=Shenzhen Research Institute of Big Data/OU=Learning Analytics/CN=10.20.20.12"

    The common name of SSL cert must match Nginx's server_name configuration.

    After you've done, run

    sudo systemctl restart nginx

The server should be available now.

Step 1 - Install Nginx, MySQL and Python

First make sure you are in your home directory and then upgrade the package manager:

cd
sudo sed -Ei 's/^# deb-src/deb-src/' /etc/apt/sources.list
sudo apt update
sudo apt upgrade

The second command prepares apt to install Python later.

Next, install Nginx and MySQL server:

sudo apt install nginx mysql-server libmysqlclient-dev

You will be prompted to set root password for MySQL. Choose a long and strong password consisting of upper and lower case letters, digits, and underscores. Avoid using other special characters. Write down this password for later use.

Verify that the installation succeeded:

systemctl status nginx mysql

Two active (running) in the output indicate success.

Installing Python is more complicated since we have to build it from source code.
Fetch the newest source code, decompress it and cd into the new folder:

Replace the URL, filename, and folder name with the latest version found on www.python.org

wget https://www.python.org/ftp/python/3.7.0/Python-3.7.0.tgz
tar xvf Python-3.7.0.tgz
rm Python-3.7.0.tgz
cd Python-3.7.0

Next, install necessary tools and libraries:

sudo apt install build-essential
sudo apt-get build-dep python3.5

As suggested by Python's official dev guide, since Ubuntu 16.04 LTS only contains Python up to version 3.5, we use build-dep python3.5. Also note that it is apt-get, not apt in the last command.

Now make sure you are in the decompressed folder, build and install Python from source:

./configure --enable-optimizations
make
sudo make altinstall

This takes a long time (because of optimizations). You can see many texts flashing in a short time and the screen may pause for a long break. You may even see Python exceptions - it's normal, don't do anything. If you accidentally interrupt a process (Ctrl + C), you have to repeat that one and its following commands.

When the installation completes, you'll see $ again. Now clean installation files and Step 1 is done.

cd
sudo rm -rf Python-3.7.0.tgz Python-3.7.0

Step 2 - Upload and Create Environment

Assume that you have uploaded decode_website.zip to your home directory. (Probably with scp command)
Decompress the archive:

cd
sudo apt install zip unzip
unzip decode_website.zip
rm decode_website.zip

Create a Python virtual environment for the application:

cd ~/decode_website
python3.7 -m venv decode_venv
source ./decode_venv/bin/activate
pip install -r requirements.txt

Next, put configuration files in place:

sudo ln ~/decode_website/deployment_configs/gunicorn.service /etc/systemd/system/gunicorn.service
sudo ln ~/decode_website/deployment_configs/gunicorn.socket /etc/systemd/system/gunicorn.socket
sudo ln -s ~/decode_website/deployment_configs/nginx_decode /etc/nginx/sites-available/nginx_decode
sudo ln -s ~/decode_website/deployment_configs/nginx_decode /etc/nginx/sites-enabled/nginx_decode
sudo rm /etc/nginx/sites-enabled/default

Step 3 - Configuration

  • Configure MySQL
    First, secure MySQL installation by running:

    mysql_secure_installation

    You'll prompted for root password, which you should have already written down. Proceed with answers:

    setup VALIDATE PASSWORD plugin: Y
Please enter 0 = LOW, 1 = MEDIUM and 2 = STRONG: 2
Change the password for root: N
Remove anonymous users: Y
Disallow root login remotely: Y
Remove test database and access to it: Y
Reload privilege tables now: Y

    Next, create database and user:

    mysql -u root -p

    In MySQL prompt, enter:

    CREATE DATABASE decode CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'decode'@'localhost' IDENTIFIED BY '-v19hVITZ3ttCMqqe89H5G_S7yE4HaoLvf-DceMMnFo';
GRANT ALL PRIVILEGES ON decode.* TO 'decode'@'localhost';
FLUSH PRIVILEGES;
\q

    You can choose a different password for user 'decode', but it should not contain special characters other than '-' or '_'. It needs to comply with strong password rule, too. You also need to change password in ~/decode_website/app_decode/config.py.

  • Configure Nginx
    Edit Nginx config file:

    sudo vi ~/decode_website/deployment_configs/nginx_decode
    1. Replace all /home/itso with your user's home folder, typically /home/username.
    2. Find two lines beginning with server_name and replace the FQDN / IP with the correct one.

    Save the file.
    Then change SSL certificate:

    1. Replace ~/decode_website/certs/cert.pem with certificate (public key) file.
    2. Replace ~/decode_website/certs/key.pem with private key file.

    Both files should be PEM format.

  • Configure Gunicorn
    Edit Gunicorn service config file: (socket config file needs no change)

    vi ~/decode_website/deployment_configs/gunicorn.service

    Replace all /home/itso with your user's home folder, typically /home/username.
    Create log files in advance to avoid permission issues:

    sudo touch /var/log/gunicorn_access.log /var/log/gunicorn_error.log
sudo chown www-data:root /var/log/gunicorn_access.log /var/log/gunicorn_error.log

  • Configure Application
    Implement the application's database table schema into MySQL and insert initial data:

    cd ~/decode_website
source ./decode_venv/bin/activate
export FLASK_APP=decode_website.py
flask db init
flask db migrate
flask db upgrade
flask initdb

  • View server logs
    For convenience, create symlink from logs to home folder:

    ln -s /var/log/decode_access.log ~/nginx_access.log
ln -s /var/log/decode_error.log ~/nginx_error.log
ln -s /var/log/gunicorn_access.log ~/gunicorn_access.log
ln -s /var/log/gunicorn_error.log ~/gunicorn_error.log

  • Restart services

    sudo systemctl daemon-reload
sudo systemctl enable gunicorn.socket
sudo systemctl enable gunicorn.service
sudo systemctl start gunicorn.socket
sudo systemctl start gunicorn.service
sudo systemctl restart nginx

The server should be available now.
